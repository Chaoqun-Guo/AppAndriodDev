## 1. 引言

### 1.1 项目背景

随着人们健康意识增强，徒步、骑行、越野跑等户外运动流行。目前主流运动App对复杂户外场景（离线地图、轨迹纠偏、环境安全）支持不足。本项目拟开发一款专注**户外运动记录**的Android应用。

### 1.2 项目目标

- 提供精准的GPS运动轨迹记录与回放
- 支持离线地图与轨迹导航
- 记录运动核心数据（距离、配速、海拔、心率等）
- 提供户外安全特性（轨迹分享、紧急联系人）

### 1.3 目标用户

- 户外徒步/登山爱好者
- 城市跑步/骑行人群
- 轻度探险与露营用户

## 2. 功能性需求

### 2.1 运动记录模块

| 编号  | 功能         | 描述                                                               |
| ----- | ------------ | ------------------------------------------------------------------ |
| FR-01 | 开始运动     | 选择运动类型（徒步/骑行/跑步/登山），点击开始记录                  |
| FR-02 | GPS轨迹记录  | 每秒记录经纬度、时间、海拔、瞬时速度、精度                         |
| FR-03 | 实时数据面板 | 显示：里程、用时、平均配速、当前海拔、累计爬升、心率（若连接设备） |
| FR-04 | 暂停/继续    | 支持运动中暂停（不记录轨迹）                                       |
| FR-05 | 结束运动     | 保存轨迹、总统计、生成摘要卡片                                     |
| FR-06 | 自动暂停     | 速度低于阈值（如0.5m/s）超过10秒，自动暂停记录                     |

### 2.2 地图与导航模块

| 编号  | 功能         | 描述                                                 |
| ----- | ------------ | ---------------------------------------------------- |
| FR-07 | 底图切换     | 支持：标准地图、卫星地图、地形图（离线包）           |
| FR-08 | 离线地图下载 | 按矩形区域或城市下载离线地图数据                     |
| FR-09 | 轨迹导航     | 导入GPX/KML文件，在地图上显示参考轨迹                |
| FR-10 | 偏航提醒     | 距离参考轨迹超过20米，震动+通知提醒                  |
| FR-11 | 位置标记     | 长按地图添加标记点（水源/营地/危险区），支持拍照备注 |

### 2.3 数据统计与分析

| 编号  | 功能         | 描述                                                               |
| ----- | ------------ | ------------------------------------------------------------------ |
| FR-12 | 运动记录列表 | 按时间倒序展示历史运动，包含缩略地图及核心指标                     |
| FR-13 | 运动详情页   | 展示：轨迹地图（可播放动画）、分公里配速表、海拔心率曲线、天气数据 |
| FR-14 | 周/月/年统计 | 运动总时长、总距离、累计爬升、卡路里估算                           |
| FR-15 | 成就系统     | 勋章：首次10km、累计爬升1000m等                                    |

### 2.4 户外安全模块

| 编号  | 功能         | 描述                                                            |
| ----- | ------------ | --------------------------------------------------------------- |
| FR-16 | 轨迹分享     | 生成轨迹图片/链接，分享到微信/QQ/朋友圈                         |
| FR-17 | 实时位置共享 | 运动中生成临时链接，好友可网页查看实时位置                      |
| FR-18 | 一键求助     | 运动中长按“求助”按钮，自动向紧急联系人发送当前位置短信+轨迹链接 |
| FR-19 | 离线安全提醒 | 当检测到无网络信号且用户偏航>200米，提示“可能处于无信号区域”    |

### 2.5 个人与设备

| 编号  | 功能         | 描述                                    |
| ----- | ------------ | --------------------------------------- |
| FR-20 | 用户档案     | 身高、体重、年龄、运动偏好              |
| FR-21 | 蓝牙设备连接 | 支持ANT+ / 蓝牙心率带、踏频器、功率计   |
| FR-22 | 数据导入导出 | 导出运动为GPX/FIT文件，支持导入外部轨迹 |

## 3. 非功能性需求

### 3.1 性能要求

| 项目              | 指标                                  |
| ----------------- | ------------------------------------- |
| GPS冷启动定位时间 | < 30秒（开阔地带）                    |
| 轨迹记录功耗      | 连续记录5小时，耗电 < 30%（主流机型） |
| 地图滑动帧率      | ≥ 50fps                               |
| 轨迹保存耗时      | < 500ms（最长10小时轨迹）             |
| 离线地图加载      | 10MB级瓦片加载 < 1秒                  |

### 3.2 可靠性

- **轨迹不丢失**：App被系统杀死时，未保存轨迹须在下次启动时提示恢复
- **GPS弱信号处理**：隧道/峡谷中自动保留最后有效点，不产生飞线

### 3.3 可用性

- 运动中主要按钮（暂停/结束）尺寸 ≥ 48dp
- 支持深色模式（户外省电且防眩光）
- 语音播报：每公里/每爬升100米播报一次

### 3.4 Android兼容性

- 最低支持：Android 10 (API 29)
- 目标版本：Android 14 (API 34)
- 适配权限：后台位置、物理活动识别、通知、相机、存储

## 4. 数据需求

### 4.1 本地存储（主要）

由于户外常无网络，优先采用SQLite + 文件系统：

- **运动记录表**：id, 运动类型, 开始时间, 结束时间, 总距离, 总爬升, 轨迹文件路径
- **轨迹点表**：运动id, 时间戳, 纬度, 经度, 海拔, 心率, 速度
- **标记点表**：运动id, 类型, 纬度, 经度, 照片路径, 备注文字

### 4.2 云端同步（可选后期）

- 支持登录后备份运动记录
- 轨迹文件上传OSS，减少本地占用

## 5. 接口需求

### 5.1 外部服务

- **地图SDK**：高德地图 / 天地图（需申请Android Key）
- **离线地图**：使用MapLibre GL Native + 矢量瓦片
- **海拔修正**：调用SRTM / ALOS DEM服务（可选）

### 5.2 硬件调用

- GPS（通过LocationManager或FusedLocationProvider）
- 加速度计（用于计步辅助）
- 气压计（若有，用于海拔精度提升）

## 6. 用户界面原型说明

### 6.1 主要页面

1. **运动首页**：大圆“开始”按钮 + 最近一次运动摘要
2. **运动中界面**：
   - 上半部分：地图（显示走过的轨迹）
   - 下半部分：滚动数据卡片
   - 底部：暂停、结束、标记三个大按钮
3. **历史列表页**：卡片式列表 + 年度统计头图
4. **我的页**：个人数据、连接的设备、离线地图管理、紧急联系人设置

### 6.2 关键交互

- 长按地图任意点 → 弹出添加标记点
- 运动中左滑数据卡片 → 显示详细图表（心率/配速曲线）
- 结束时自动生成运动摘要，提供“保存”与“丢弃”选项

## 7. 异常场景处理

| 场景                | 处理方式                                        |
| ------------------- | ----------------------------------------------- |
| 运动开始时GPS未定位 | 倒计时提示“等待定位中...”，可强制开始（仅计步） |
| 运动中接电话        | 后台继续记录，返回时恢复地图更新                |
| 手机电量不足15%     | 提示“是否进入省电模式”（降低定位频率至5秒/次）  |
| 存储空间不足        | 警告并建议导出历史轨迹至外部存储                |

## 8. 技术选型建议

| 层       | 技术                                              | 理由                               |
| -------- | ------------------------------------------------- | ---------------------------------- |
| 语言     | Kotlin                                            | Android官方推荐，协程方便处理GPS流 |
| UI框架   | Jetpack Compose                                   | 声明式UI，地图与数据绑定简便       |
| 定位     | FusedLocationProviderClient + 系统LocationManager | 兼顾省电与精度                     |
| 数据库   | Room + SQLite                                     | 存储轨迹点与元数据                 |
| 地图渲染 | MapLibre GL Native                                | 开源、支持离线矢量瓦片             |
| 轨迹解析 | gpx-parser + geotools（精简）                     | 支持GPX/KML                        |
