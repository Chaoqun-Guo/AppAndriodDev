# Outdoor运动记录 App

Android 端户外运动记录应用（Kotlin + Jetpack Compose）。

## 当前功能

- 运动页
  - 运动类型选择与开始按钮位于页面下半区。
  - 记录页支持开始/暂停/继续/结束。
  - 基于陀螺仪 + 地理位置联合判断是否在运动，静止超过阈值自动暂停。
- 历史页
  - 周统计、月统计、年统计支持日期选择器。
  - 历史记录支持按日期筛选查看。
- 我的页
  - 分为个人信息、设备、安全三个部分。
- 设置页（统一配置入口）
  - 可配置记录页展示字段（里程/用时/配速/海拔/爬升/心率/速度/运动状态）。
  - 可配置服务接入：地图服务、天气服务、轨迹共享服务。
  - 可配置备份能力：本地备份 + 云端备份（Google Cloud / S3兼容 / WebDAV）。

## 备份能力

### 备份目标

- 本地备份：将运动记录快照保存到应用私有目录 `files/backups/`。
- 云端备份：用户可自行选择并配置提供商：
  - `Google Cloud Storage`
  - `S3 兼容对象存储`
  - `WebDAV 云盘`

### 用户自配置规则

- 备份能力由用户在「设置 -> 备份配置」中自行配置。
- 云端备份必须满足以下条件才会生效：
  1. 选择云端提供商
  2. 填写必要配置（API Key、Bucket/Path）
  3. 手动打开云端备份开关
- 若用户未完成云端配置：
  - 云端备份不会执行
  - 页面不会显示“已启用云备份”相关内容

### 统一备份策略模板

应用内置统一模板，用户可直接套用并再微调：

- `安全优先（每日）`：24 小时、仅 Wi-Fi、仅充电、保留 30 天
- `均衡（每6小时）`：6 小时、仅 Wi-Fi、非必须充电、保留 14 天
- `高频（每1小时）`：1 小时、允许移动网络、非必须充电、保留 7 天
- `自定义`：用户自行修改间隔/网络/充电/保留天数

### 手动备份

- 设置页提供“立即备份”按钮。
- 执行后会记录最近一次备份时间与结果。

## 服务接入规则（API Key）

- 服务是否可用由两项共同决定：
  1. 用户提供对应 `API Key`
  2. 用户在设置中手动开启该服务
- 若未提供 API Key：
  - 该服务不会启用
  - 页面中不会显示该服务相关内容

## 使用说明

1. 打开「设置」页
2. 选择记录页展示字段
3. 如需接入外部服务，填写对应 API Key 并开启服务开关
4. 如需启用备份，在备份区选择目标、填写云端配置并选择策略模板
5. 返回运动页/我的页查看配置效果，或直接执行“立即备份”

```bash
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
./gradlew :app:assembleDebug
```
