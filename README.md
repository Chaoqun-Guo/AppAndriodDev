# Outdoor运动记录 App

Android 端户外运动记录应用（Kotlin + Jetpack Compose）。

## 当前功能

- 底部导航
  - 固定保留 4 个页面：数据面板、运动、记录、我的。
  - 底部导航顺序支持在设置中自定义调整（上移/下移/重置）。
  - 全站采用卡片式布局风格。
- 运动页
  - 运动类型选择与开始按钮位于页面下半区。
  - 页面结构优化为主流户外App信息层级：状态速览 -> 开始新活动操作区。
  - 运动页已移除“本周目标设定/最近一次活动”卡片，减少干扰并聚焦开练操作。
  - 记录页支持开始/暂停/继续/结束。
  - 开始运动时可开启路径记录；开启后本次运动会自动产出路径记录（用于后续回溯）。
  - 路径记录开关位于“开始运动”按钮下方，便于起练前快速确认。
  - 点击开始后，运动页会同步显示“运动中”并锁定页面操作，避免误触。
  - 基于陀螺仪 + 地理位置联合判断是否在运动，静止超过阈值自动暂停。
- 记录页轨迹区域可在设置中自定义显示/隐藏。
  - 记录页布局优化为主流“运动中”结构：状态摘要+核心三指标（里程/用时/均配）+传感器状态+地图主视图+底部控制按钮。
  - 记录页显示陀螺仪/方位/指南针/海拔传感器工作状态（红/黄/绿）。
  - 记录页轨迹区域已接入真实轨迹渲染（Canvas 折线轨迹，起终点标记）。
  - 记录页操作区提供 4 按钮：继续、标记、暂停、结束。
  - 记录页移除“路径记录与回溯”模块，聚焦实时记录操作。
- 数据面板页
  - 新增独立实时数据面板页面。
  - 顶部天气卡默认基于手机地理位置自动更新。
  - 已移除天气查询功能，仅保留定位驱动天气。
  - 最近三天预报默认折叠，按需展开查看。
  - 天气信息扩展为户外场景关键字段：温度、体感、降水、湿度、气压、风速/阵风/风向、UV、能见度、云量、露点、日出日落、更新时间。
  - 三日预报新增：风向、阵风、能见度。
  - 海拔高度与经纬度同一行展示。
  - 海拔卡宽度约 1/3，经纬度卡约 2/3，并保持等高。
  - 指南针与水平仪同一行展示。
  - 指南针卡与水平仪卡保持等高。
  - 经纬度卡片增加地理位置信息（逆地理解析地址）。
  - 经纬度卡同步展示地图来源（手机地图服务 / 外部地图接口）。
  - 提供水平仪功能（Pitch/Roll + 气泡水平指示）。
  - 非运动状态也会持续刷新传感器与定位数据。
  - 支持自定义数据刷新频率（秒）。
  - 提供指南针自动校准开关 + 手动校准模式（设当前方向为北/重置校准）。
  - 数据面板聚焦传感器与定位展示，不承载路径回溯操作。
  - 海拔传感器数据可自动更新并实时显示。
- 历史与统计（已整合进“我的”页面）
  - 历史页采用卡片流布局：运动类型筛选 -> 自定义时间统计区间 -> 日/周/年统计（同一行各 1/3）-> 最近记录列表。
  - 日历点选后，会联动更新统计卡片。
  - 支持自定义时间范围统计（开始/结束日期）。
  - 历史记录支持按运动类型筛选（徒步/骑行/跑步/登山）。
  - 历史记录卡片字段支持自定义展示（距离/时长/爬升/轨迹点）。
  - 历史默认展示最近 5 条记录，支持「更多」逐步加载。
  - 点击具体记录后跳转到独立详情页，展示活动详情分析：轨迹图、配速曲线、分段配速。
  - 活动详情内提供路径回溯模块（正向/反向回溯入口）。
  - 详情页地图支持长按选锚点：按 Haversine 计算最近轨迹点，超 30m 提示“附近无轨迹点，请长按轨迹线附近”。
  - 锚点选中后显示高亮标记与回溯控制面板（锚点里程、海拔、速度/配速、轨迹相对进度）。
  - 支持正向回溯（绿色）与反向回溯（橙色）动画播放，可播放/暂停、0.5x/1x/2x/4x 倍速、方向即时切换。
  - 支持拖动进度条跳转任意轨迹位置，拖动后自动暂停并同步地图与数据。
  - 轨迹点不足 2 个时禁用回溯并提示；超长轨迹（>20000点）每帧跳点保证流畅。
- 我的页
  - 删除设备卡片，保留共享与安全、历史、设置模块，并支持顺序自定义。
  - 共享与安全支持“共享联系人邮箱”增删改查（多邮箱列表 + 保存）。
- 设置（已整合进“我的”页面，统一配置入口）
  - 可配置记录页展示字段（里程/用时/配速/海拔/爬升/速度/运动状态）。
  - 可配置轨迹区域显隐。
  - 可配置数据面板刷新频率与指南针自动校准开关。
  - 可配置“我的”页面模块显示顺序。
  - 设置页采用折叠式分组交互（含导航顺序、服务接入、备份等）。
  - 可配置服务接入：地图服务（手机地图优先 + 外部接口保留）、天气服务、轨迹共享服务。
  - 新增 SMTP 邮件共享配置（Host/Port/账号/密码/发件人/TLS）；用户自行配置完成后可启用。
  - 可配置备份能力：本地备份 + 云端备份（Google Cloud / S3兼容 / WebDAV）。
  - 不提供广告、订阅或会员分级；仅在设置页末尾提供「赞助支持」入口（自愿赞助，不影响功能）。

## 架构优化（模块解耦）

- 已将核心业务计算从 `AppViewModel` 拆分为独立 domain 模块，降低模块间耦合：
  - `domain/history/HistoryDomain.kt`：历史筛选与日/周/月/年/自定义统计计算
  - `domain/weather/WeatherDomain.kt`：定位天气构建、三日预报构建
  - `domain/share/SmtpEmailSender.kt`：SMTP 邮件发送能力（用户自配置）
  - `domain/replay/ReplayDomain.kt`：详情回溯点构建、回放步进策略、Haversine距离计算
- `AppViewModel` 仅负责状态编排与模块协作，业务规则由 domain 层统一实现，便于后续单元测试与功能扩展。
- UI 第二阶段拆分已完成（页面级解耦）：
  - `ui/screens/HomeScreen.kt`
  - `ui/screens/RecordingScreen.kt`
  - `ui/screens/DataPanelScreen.kt`
  - `ui/screens/ProfileScreen.kt`（我的页与设置页独立模块）
  - `ui/screens/HistoryScreen.kt`（历史页与活动详情回溯分析独立模块）
  - `ui/common/Formatters.kt`（里程/时长/配速/时间/方位统一格式化）
  - `ui/common/RecordingUi.kt`（记录页状态文案、指标构建、传感器状态点统一组件）
  - 历史页通过 `historyContent` 插槽注入，降低 Profile 模块与历史实现的耦合
  - History 模块仅暴露必要回调，移除未使用接口参数，进一步收敛耦合面
  - `MainActivity` 主要负责导航与页面装配，降低单文件复杂度。

## 备份能力

### 备份目标

- 本地备份：将运动记录快照保存到应用私有目录 `files/backups/`。
- 云端备份：用户可自行选择并配置提供商：
  - `Google Cloud Storage`
  - `S3 兼容对象存储`
  - `WebDAV 云盘`

### 用户自配置规则

- 备份能力由用户在「设置 -> 备份配置」中自行配置。
- 云端备份必须满足以下条件才会生效：
  1. 选择云端提供商
  2. 填写必要配置（API Key、Bucket/Path）
  3. 手动打开云端备份开关
- 若用户未完成云端配置：
  - 云端备份不会执行
  - 页面不会显示“已启用云备份”相关内容

### 统一备份策略模板

应用内置统一模板，用户可直接套用并再微调：

- `安全优先（每日）`：24 小时、仅 Wi-Fi、仅充电、保留 30 天
- `均衡（每6小时）`：6 小时、仅 Wi-Fi、非必须充电、保留 14 天
- `高频（每1小时）`：1 小时、允许移动网络、非必须充电、保留 7 天
- `自定义`：用户自行修改间隔/网络/充电/保留天数

### 手动备份

- 设置页提供“立即备份”按钮。
- 执行后会记录最近一次备份时间与结果。

### 历史备份文件管理

- 设置页支持查看历史备份文件列表。
- 支持打开备份文件并直接编辑内容后保存（用于手工修正备份元数据）。

## 服务接入规则（API Key）

- 服务是否可用由两项共同决定：
  1. 用户提供对应 `API Key`
  2. 用户在设置中手动开启该服务
- 若未提供 API Key：
  - 该服务不会启用
  - 页面中不会显示该服务相关内容

## 使用说明

1. 打开「我的 -> 设置」页
2. 选择记录页展示字段
3. 如需调整底部导航顺序，在设置中使用“上移/下移/重置”
4. 如需接入外部服务，填写对应 API Key 并开启服务开关
5. 如需启用备份，在备份区选择目标、填写云端配置并选择策略模板
6. 返回运动页/我的页查看配置效果，或直接执行“立即备份”

## 路径回溯说明

1. 在「运动」页开始前，可选择“开始运动时开启路径记录”。
2. 回溯执行入口位于「我的 -> 历史 -> 活动详情 -> 路径回溯模块」。
3. 路线来源：
   - 实时轨迹
   - 历史已存盘回溯路线
4. 支持“反向回溯 / 正向回溯”切换。
5. 默认会定位到最近记录节点作为回溯目标，也可手动选择任意节点。
6. 支持路径偏离提醒（路径偏离距离 + 指针偏离提示），用于提示回溯状态。
7. 可手动将当前轨迹保存为路径记录；运动保存时若开启路径记录也会自动写入。
